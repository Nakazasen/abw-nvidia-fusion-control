# NON_CLAIMS

## Purpose

Make explicit what must not be claimed until governance promotes readiness.

## Ownership

- Owner: Governance Recorder + Auditor
- Canonicality: Canonical non-claims boundary document

## SOURCE OF TRUTH

- Current readiness and promotion context: `03_OPERATIONS/HANDOFF.md`, `05_DECISIONS/ROADMAP.md`, `05_DECISIONS/DECISIONS.md`.

## Active Non-Claims

Do not claim:
- `DAILY_USE_READY`
- production-ready
- full bridge ready
- Cognitive OS achieved
- enterprise-grade security
- packaging-ready
- VS Code parity
- Cursor parity

## Interpretation Rule

- Absence of blocker in one path is not equivalent to global readiness.
- Any readiness upgrade requires explicit governance decision and cross-file update.

## Update Rule

- If a non-claim is retired, record decision in `05_DECISIONS/DECISIONS.md` first.
- Then update `03_OPERATIONS/HANDOFF.md`, `05_DECISIONS/ROADMAP.md`, and `05_DECISIONS/RISKS.md` consistently.
